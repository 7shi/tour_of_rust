Representing Nothing

In other languages, the keyword null is used to represent an absense of value. It creates alot of difficulty in languages because it creates the possibility our program might fail when interacting with a variable/field.

Rust does not have null. But it is not ignorant to the importance of representing nothing! Consider a naive representation using a tool we already know.

This pattern of providing a None alternative representation for one or many values is so common Rust because of it's lack of a null value. Generics help solve this challenge.
